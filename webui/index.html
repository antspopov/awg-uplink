Настройка интерфейсов

Настройка сетевых параметров сервера.
Egress интерфейс
не применено
Egress — основной выход в интернет с маршрутом по умолчанию. Если шлюз не указан, будет использовано системное значение.
Новые входящие на egress разрешены только на указанных портах (остальное — drop). SSH по умолчанию 22.
Только правила с меткой awg-web-ui-fw — остальные порты в UFW не меняются. Нужны активный UFW (sudo ufw enable) и пакет ufw. Georouting и DNS-lock по-прежнему на nftables.
Ingress интерфейс
не применено
Ingress — дополнительный интерфейс/адрес для подключения пользователей к серверу из интернета. Если дополнительного интерфейса/адреса на сервере нет, настраивать ingress не нужно — роль ingress возьмет на себя Egress интерфейс.
Когда ingress совпадает с egress, порты объединяются в одном правиле. Отдельная карточка ingress — только если выбран другой интерфейс/адрес.
Проверьте параметры перед сохранением.

Dnsmasq и dnscrypt-proxy
Списки доменов: не обновлялись
Нужна для снижения риска утечки DNS и чтобы запросы клиентов AmneziaVPN к контейнеру AmneziaDNS проходили через ваш dnsmasq/dnscrypt на шлюзе docker-сети (обычно 172.29.172.1). При режиме маршрутизации по спискам доменов (georouting) опция включена и заблокирована: без неё доменная схема работает ненадёжно.
В приложении Amnezia VPN на сервере должен быть установлен сервис AmneziaDNS (контейнер): Settings → Servers → Ваш сервер → Services → AmneziaDNS → Install. Также необходимо включить пересылку DNS: Settings → Connections → Use AmneziaDNS. Тогда клиенты шлют DNS в контейнер, а мы перенаправляем его на хостовый dnsmasq/dnscrypt.
Фоновый сервер проверяет конфигурацию примерно каждые 30 с; после подмены контейнер AmneziaDNS перезапускается только если действительно изменился форвард корня «.» (не на каждом цикле).
Использовать с осторожностью: на некоторых клиентах возможны сбои (жёстко прошитые резолверы, приложения с собственным DNS).
На шлюзе включается перенаправление UDP/TCP 53 в Docker-сети Amnezia на Dnsmasq (172.29.172.1 или фактический Gateway сети), блокируется DoT (853) и HTTPS к типичным публичным DoH. Так проще удержать запросы на вашем Dnsmasq/dnscrypt, снизить утечки DNS и сделать стабильнее режим «По списку доменов» в геомаршрутизации.
Опция самостоятельна: не связана с подменой AmneziaDNS и режимами геомаршрутизации; действует по флагу в dns.json после «Применить» или загрузки юнита при старте системы.
Домены из защищённых готовых списков всегда уходят в dnscrypt; остальные готовые списки и include/exclude — при включённом режиме «по доменам» в georouting. Обновление списков — каждые 6 ч или по «Применить».

В некоторых сетях РФ установка MTProto может завершаться ошибкой из-за ограничений доступа к внешним ресурсам.
Рекомендуется заранее настроить туннельный интерфейс с выходом через другую страну и временно переключить маршрут по умолчанию в режим В тунельный интерфейс.
